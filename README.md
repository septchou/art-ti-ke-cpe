#  Art-ti-ke 

Art-ti-ke เป็นเว็บไซต์ที่ให้บริการเกี่ยวกับการ commission งาน Digital Art โดยเชื่อมโยงระหว่างศิลปินและลูกค้าเพื่อการซื้อขายที่สะดวกสบายและปลอดภัยต่อกันและกัน ช่วยให้ลูกค้าค้นพบศิลปินที่ตรงกับความต้องการและ commission ได้ง่ายขึ้น รวมถึงทางฝั่งศิลปินเองที่สามารถถูกค้นพบได้ง่ายขึ้น

---

## Design Process

### User Research
ทางผู้จัดทำได้ทำการสังเกตุระบบการซื้อขาย commission งานศิลปะ Digital art มานานพอสมควร ทั้งปัญหาจากคนใกล้ตัว และปัญหาจากนักวาดอื่น ๆ ที่ติดตามผลงาน ทำให้ได้ข้อสรุปมาดังนี้
 1. ระบบ Algorithm จากเว็บไซต์ต่าง ๆ
ปัญหานี้เป็นปัญหาที่คาราคาซังมานาน ในหลายแอพพลิเคชั่นนั้น แม้ผู้ใช้จะมีผู้ติดตามมาก แต่ตัวงานของเขา กระทั่งข่าวคราวการเปิดรับ commission เองก็ไม่ถูกแจ้งเตือนให้กับผู้ติดตาม ยังไม่นับนักวาดที่ใหม่ต่อวงการที่กว่าจะถูกค้นพบงานวาดตัวเองและถูกยอมรับ ก็ใช้เวลาไปมากโขจนหมดกำลังใจและเลิกทำงานสายนี้ไป จึงคิดเห็นว่าควรมีเว็บไซต์ที่รวบรวมศิลปินนักวาดไว้เพื่อให้ถูกค้นพบได้ง่ายขึ้น รวมถึงลูกค้าเองก็จะสามารถหานักวาดที่ตรงกับความต้องการได้ง่ายขึ้นมากเช่นกัน

 2. ปัญหาการโกงและลิขสิทธิ์
เนื่องจากการซื้อขายส่วนใหญ่นั้น ซื้อขายผ่านช่องทางโซเชียลมีเดียและตกลงกันปากเปล่า ทำให้เกิดปัญหามากมาย ทั้งการสั่งคอมมิชชั่นแล้วไม่จ่ายเงิน การไม่ส่งงานตามกำหนดเวลา การรับเงินแล้วหนีหาย การตกลงลิขสิทธิ์ ซึ่งไม่สามารถแก้ไขได้ง่ายและเกิดขึ้นบ่อยมาก ๆ หากมีแพลตฟอร์มที่สามารถจัดการปัญหานี้ได้ การซื้อขายก็จะมีความปลอดภัยมากขึ้นจากทั้งตัวศิลปินและลูกค้า

### 3. **Artist Profile (โปรไฟล์ศิลปิน)**
ผู้ใช้สามารถดูโปรไฟล์ของศิลปิน รวมถึงตัวอย่างผลงานที่ผ่านมา, ข้อมูลเกี่ยวกับศิลปิน และคอมมิชชั่นที่กำลังเปิดรับ

- แสดงรายละเอียดเกี่ยวกับศิลปิน (information) เช่น ชื่อแอคเคาท์ ข้อมูลส่วนตัว ช่องทางติดต่อ

### 4. **Commission Request (การขอคำสั่งงาน)**
ลูกค้าสามารถสร้างคำขอสำหรับคอมมิชชั่นที่ต้องการให้ศิลปินทำได้โดยง่าย

- ฟอร์มการกรอกข้อมูลรายละเอียดเกี่ยวกับงาน (เช่น ขนาด, รูปแบบ, ลักษณะเฉพาะ, แท็กประเภทงาน)
- ระบบการตั้งราคาเบื้องต้น ระยะเวลาที่ต้องการ และอัปโหลดตัวอย่างที่ต้องการ (ตัวอย่างเป็น optional)
- ระบบการแจ้งเตือนเมื่อศิลปินรับงาน

### 5. **Wallet (การชำระเงิน)**
ระบบการชำระเงินที่ปลอดภัยและสะดวก ใช้วิธี Top-up เงินใน wallet เพื่อเช็คยอดเงินและชำระเก็บไว้ในระบบ คืนเงินเมื่อถูกยกเลิกคอมมิชชั่น และจ่ายเงินให้ศิลปินเมื่อได้รับงานคอมมิชชั่นแล้ว รวมถึงฝั่งศิลปินที่ต้องชำระเงินจำนวนหนึ่งเพื่อเป็นค่าประกัน

- การชำระเงินออนไลน์ผ่านบัตรเครดิต/เดบิต หรือช่องทางการชำระอื่น ๆ
- ระบบการเก็บเงินเพื่อให้มั่นใจว่าศิลปินจะได้รับค่าจ้างเมื่อถึงเวลา
- ค่าปรับและค่าชดเชยเมื่อศิลปินยกเลิกการว่าจ้างกลางคัน โดยหากศิลปินเป็นผู้ยกเลิก (เช่น ไม่ส่งงานเมื่อครบกำหนดเวลา) ศิลปินจะต้องเสียค่าชดเชย

### 6. **Manage Requests (การจัดการคำขอ)**
ลูกค้าสามารถจัดการคำขอของตัวเองได้ เช่น การสร้างคำขอ การยกเลิกคำขอ การติดตามคำขอ และศิลปินสามารถจัดการคำขอจากลูกค้าได้เช่นกัน เช่น การยอมรับคำขอ การปฏิเสธคำขอ

- ระบบให้คะแนนตามผลงาน
- ระบบแสดงความคิดเห็นจากลูกค้า

### 7. **FAQ (คำถามที่พบบ่อย)**
หน้า FAQ ที่ให้ข้อมูลเกี่ยวกับกระบวนการ commission, การชำระเงิน, การติดต่อศิลปิน ฯลฯ

---

### 👨‍💻 **For Developers**  

#### Prerequisites
- Unity Editor (Compatible with your project version)
- Firebase SDK for Unity, Version 12.4
- A Firebase account with access to Firebase Console

#### 1️⃣ **Install Unity**  
- Download Unity Hub from [Unity Download](https://unity.com/download).  
- Use Unity Version: **<2022.3.46f1>**.  

#### 2️⃣ **Set Up Firebase SDK**  
- Download the **Firebase Unity SDK** from [Firebase Unity SDK](https://firebase.google.com/docs/unity/setup).  
- Download version **12.4** of the SDK.
- Choose the `.unitypackage` files for **Firebase Auth** and **Firebase Database** from the SDK package.
- nsure all necessary files are selected and click **Import**.

#### 3️⃣ **Configure Realtime Database and Authentication**  
1. Access the **Firebase Console**.  
2. Enable **Realtime Database**:  
   - Go to Database > Create Database.  
3. Enable **Authentication**:  
   - Go to Authentication > "Sign-in Method" > Enable Email/Password.  

#### 4️⃣ **Add Config Files to Unity**  
- Download `google-services.json` (for Android) and `GoogleService-Info.plist` (for iOS).  
- Place these files in the **Assets/Firebase** directory.
